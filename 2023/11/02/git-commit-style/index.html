Git提交规范

前言

仅仅用于自己的一些比较大的项目，或者说提交量比较多的项目中， 尽可能遵守这里记录的提交风格，不然我自己都看不懂提交了些啥玩意。

博客文章提交

基本格式：<type>(type, type): <msg>。其中的type为以下类型：

类型 意义
draft 创建新的草稿
post 创建新文章
corr 文章错误更正
theme 主题更新
config 配置更新
update 文章内容更新
others 其他操作

<type>表示此次提交的主要操作，(type)表示此次提交中还进行其他附加操作。 <msg>描述主要操作都干了啥， 对于其他操作应该有比较详细的描述， 对于附加操作可以不说明。

自己项目的代码提交

基本格式：<type>: <msg>。

类型 意义
feat 新功能实现
fix 缺陷修复
typo 错别字更正
revert 版本回退
style 代码风格变更
refact 代码重构
docs 修改注释
merge 代码合并
others 其他操作
